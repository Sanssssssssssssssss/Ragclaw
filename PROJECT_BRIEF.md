# PROJECT_BRIEF

## 项目名称
RAG_Model

## 项目目标
基于开源仓库 `lyxhnu/Skill-First-Hybrid-RAG` 搭建一套可在本地运行、可持续演进、以私有仓库为主仓的 RAG 学习与改造项目。

本项目的核心目标分为三层：

1. 先完成一个与上游仓库能力接近的可运行基线版本。
2. 在可运行基线上持续学习并演进不同 RAG 方向，例如多模态 RAG、GraphRAG / RAGGraph、评测与记忆机制改造。
3. 将项目目标、需求、架构、决策、任务和当前状态外部化到 Markdown 文档中，支持多轮会话下的稳定协作。

## 背景与动机
- 参考对象是一个 `Skill-first + hybrid retrieval + LangGraph + Web UI` 的 RAG 系统。
- 我们希望保留其“本地优先、文件可审计、技能可编辑、检索可观测”的优点。
- 我们需要一个自己的私有代码仓库，作为后续持续更新、实验和知识沉淀的主存储。

## 当前范围
### 范围内
- 建立私有项目仓库和项目记忆机制。
- 复现上游仓库的基础结构与最小可运行能力。
- 支持本地启动前后端并完成基础聊天与检索验证。
- 为未来引入多模态 RAG、GraphRAG / RAGGraph、评测模块预留演进路径。

### 暂不纳入
- 生产级多租户、鉴权、计费、SaaS 化部署。
- 复杂云基础设施编排。
- 在需求未明确前大规模重写上游系统。

## 成功标准
- 私有仓库建立完成，并能承载后续持续开发。
- 项目文档能够独立描述目标、需求、状态、决策与任务，不依赖聊天上下文。
- 第一阶段可以在本地启动前端与后端，并完成一次端到端验证。
- 第二阶段可以在不破坏基线的前提下增加新的 RAG 研究分支或模块。

## 非目标
- 不是一开始就做通用型企业知识平台。
- 不是一开始就追求最复杂、最前沿的多模态或图谱架构。
- 不以“尽快重写全部代码”为目标。

## 关键约束
- 所有实现都应能映射到 `REQUIREMENTS.md`。
- 每轮协作都应优先以项目记忆文件重建上下文。
- 默认采用简单、稳健、可维护方案，避免过度设计。
- 以私有仓库作为我们的主线工作区；上游仓库作为参考或同步来源。

## 已知假设
- “GitHub page 上但是 private” 当前先按“GitHub 私有仓库”理解，而不是公开静态站点 GitHub Pages。
- 第一阶段优先支持 Windows 本地开发环境。
- 第一阶段以复现上游 README 所示基础能力为主，再讨论结构性改造。

## 待确认问题
- 私有仓库策略采用 `fork`、`mirror` 还是“新建私有仓库后手动引入上游代码”。
- 第一阶段默认模型供应商与 API Key 来源。
- 我们希望对上游做“尽量保持兼容的增量改造”还是“尽快抽象出新的实验层”。
